Beginnings Schools Website - Admin Guide

This guide explains how to update common content on the website without needing technical knowledge.

1. Updating Current Openings

The current openings displayed on the website are controlled by a single file: data/openings.json

How to Update

  1. Open data/openings.json in a text editor
  2. Find the location and program you want to update
  3. Change the status, statusText, and note values
  4. Update the _lastUpdated date at the top of the file
  5. Save the file

Status Options

Status Value Display When to Use
waitlist Waitlist Only No immediate openings; families join the waitlist
limited Limited Openings A few spots available within 30-60 days
enrolling Now Enrolling Actively accepting new families for upcoming term
closed Not Accepting Program at capacity, not taking waitlist

Example Change

To change Queen Anne Toddlers from "Limited Openings" to "Waitlist Only":

"toddlers": {
  "status": "waitlist",
  "statusText": "Waitlist Only",
  "note": "Openings expected Summer 2025.",
  "badgeColor": "orange"
}
Important: Be careful with quotation marks and commas. JSON is sensitive to formatting. If the website breaks after your edit, check that all quotes are closed and commas are in the right places.

2. Updating Testimonials

Testimonials are stored in data/content.json under the testimonials section. Note: Testimonials are also hardcoded on the homepage for fallback - you may want to update both.

Testimonial Format

{
  "quote": "The testimonial text goes here...",
  "author": "Parent Name",
  "details": "Parent of a 3-year-old, Queen Anne",
  "location": "queenAnne"
}

3. Updating Team Bios

Team information is stored in data/content.json and also in the HTML files. For full updates, you'll need to edit both:

  • data/content.json - for any dynamic uses
  • team.html - for the main team page

4. Updating Contact Information

Location details (address, phone, email) are in data/site-config.json and also in the footer of each HTML page. Update both for consistency.

5. Adding/Replacing Photos

  1. Add new images to the images/ folder
  2. Use descriptive filenames (e.g., qa-classroom-art-2024.jpg)
  3. Update the src attribute in the relevant HTML file
  4. Always include a descriptive alt attribute for accessibility

Recommended Image Sizes

Image Type Recommended Size Format
Hero images 1200 x 800 pixels JPG (optimized)
Program cards 800 x 600 pixels JPG
Team photos 400 x 400 pixels (square) JPG
Gallery images 800 x 600 pixels JPG
Tip: Use a tool like Squoosh to compress images before uploading. Smaller file sizes = faster page loads.

6. File Locations Quick Reference

What to Update File to Edit
Current openings data/openings.json
Hero text index.html and data/content.json
Testimonials index.html and data/content.json
Team bios team.html and data/content.json
Program details programs.html
Location info data/site-config.json + page footers
SEO titles/descriptions <head> section of each HTML file
Site colors/fonts css/styles.css (CSS variables at top)

7. Testing Changes

  1. Open the website in a browser after making changes
  2. Check the page where you made updates
  3. Test on mobile (use browser dev tools or your phone)
  4. If something looks broken, check your recent edits for typos

8. Getting Help

If you run into issues or need to make changes beyond what's covered here:

  • Review this guide again for the specific task
  • Check that you saved the file after editing
  • Try clearing your browser cache (Ctrl+Shift+R or Cmd+Shift+R)
  • Contact your web developer for complex changes
Remember: Always keep a backup of files before making major changes. You can copy the entire data/ folder to a safe location before editing.
